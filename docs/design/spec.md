# md2idx 設計仕様

[![MKDN](https://img.shields.io/badge/MKDN-review-red?style=for-the-badge)](https://mkdn.review/?url=https%3A%2F%2Fraw.githubusercontent.com%2Foubakiou%2Fmd2idx%2Frefs%2Fheads%2Fmain%2Fdocs%2Fdesign%2Fspec.md)

## 目的

大きな Markdown を「見出しレベルで分割した JSON」に変換する CLI ツール。
先頭に見出し一覧（目次）を持たせ、`jq` で目的のセクションだけを切り出して
LLM に渡す用途を想定する。巨大ドキュメント全体を読ませずに、必要部分だけを
最小トークンで供給することがゴール。

```
md2idx large.md | jq -r '.index'              # まず目次を読む
md2idx large.md | jq -r '.sections[2]'        # 必要なセクションを取る
md2idx large.md | jq -r '.sections[0:3][]'    # ある見出し配下をまとめて取る
```

## 出力フォーマット

```jsonc
// この spec.md 自身を md2idx にかけた場合の出力（sections は冒頭のみ抜粋）
{
  "index": "# 0. md2idx 設計仕様\n## 1. 目的\n## 2. 出力フォーマット\n### 3. 参照モデル\n## 4. 不変条件\n## 5. 設計判断\n## 6. パーサ\n## 7. エッジケースの扱い\n## 8. CLI インターフェース（暫定）\n## 9. 未決事項",
  "sections": [
    "# md2idx 設計仕様",
    "## 目的\n\n大きな Markdown を「見出しレベルで分割した JSON」に変換する CLI ツール。\n…（以下本文）",
    "…（3 以降のセクションが続く）",
  ],
}
```

- `index`: Markdown 見出しマーカー付き番号目次を表す単一文字列。LLM に読ませる目次そのもの。
  各行は `<#でレベル表現> <連番>. <見出しテキスト>`。連番は `sections` の配列添字に一致する。
- `sections`: 見出し単位に分割した「見出し行＋その本文」の生 Markdown 文字列の配列。
  **文書の出現順（深さ優先の文書順）でフラットに並べる。**

### 参照モデル

- 単体取得: 目次で番号を読み、`jq '.sections[N]'`。
- 範囲取得: 目次の見出しレベル（`#` の数）で階層を読み、ある見出し配下が占める連続範囲を
  LLM が判断して `jq '.sections[N:M]'`。
- 番号は番号パス（`1.1.2`）ではなく通し番号。毎回 md2idx を実行したスナップショットを
  使う前提のため、編集による番号ズレは問題にならない。

## 不変条件

範囲取得の正しさは次の一点に依存する。実装・テストで必ず担保する。

> **`sections` は常に文書の出現順に並ぶ。** これにより、ある見出しの子孫は
> 必ず連続した添字範囲を占める。範囲取得が成立する前提。

## 設計判断

| 論点               | 決定                                   | 根拠                                               |
| ------------------ | -------------------------------------- | -------------------------------------------------- |
| 階層表現           | フラット配列（ネストしない）           | 添字一発で取得でき jq フィルタが構造に依存しない   |
| 本文の中身         | 見出し行を含む生 Markdown 文字列       | LLM に渡す前提で情報欠落なし・実装が軽い           |
| セクション識別子   | 通し番号（＝配列添字）                 | 目次→取得が最短。スナップショット運用で安定        |
| 目次の形           | `#` マーカー付き番号の単一文字列       | LLM が最も慣れた見出し記法・空白を数えず階層が確定 |
| 範囲のスライス併記 | しない（番号のみ）                     | シンプルさ優先。範囲は目次の `#` の数から読み取る  |
| トップレベル       | `{ "index", "sections" }` オブジェクト | `.index` / `.sections` で jq から扱いやすい        |

## パーサ

外部依存ゼロの自前行スキャンパーサを採用する。

- 本ツールが必要なのは「見出しの位置検出」だけであり、フル Markdown AST は不要。
  行単位のスキャンで ATX 見出し（`# `）と setext 見出し（`===` / `---` 下線）を
  検出し、コードフェンス（` ``` ` / `~~~`）内をスキップする構成で十分。
- 外部ランタイム依存がないため、`vp build` のデフォルトバンドルだけで配布できる。

## エッジケースの扱い

- **見出し前のプリアンブル**（最初の見出しより前の本文）: 先頭に番号 0 の
  セクションとして収容する。目次では見出しの無い項目として扱う。
- **レベルスキップ**（`#` の次が `###`）: 連番は出現順で素直に採番。目次の
  `#` の数は原文の見出しレベルをそのまま反映する。
- **コードフェンス内の `#`**: 見出しと誤認しない（AST により担保）。
- **本文ゼロの見出し**: 見出し行だけのセクションとして `sections` に含める。
- **見出しテキスト中の記法**（`` `code` ``、リンク等）: 目次にはプレーン
  テキスト化して載せる（記法は剥がす）。
- **ATX 見出しの `#` 後スペース必須**: `# Title` は見出しだが `#` 単独行は
  見出しにしない。テキストのない見出しの誤検出を防ぐための意図的な制限。
- **setext H2 の `-` は 2 文字以上**: `---` は認識するが `-` 単独は認識しない。
  水平線（thematic break）との誤認を防ぐための意図的な制限。

## CLI インターフェース（暫定）

```
md2idx [file]      # 引数のファイル、無ければ標準入力を読む
        --pretty   # 整形して出力する（既定は jq に流しやすい 1 行出力）
```

- 出力は標準出力（`jq` へのパイプ前提）。
